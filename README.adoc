A multi-platform memory game built with Flutter.
Numbers are scattered across the screen; tap them in ascending order — but after the first tap, they vanish.

---

== Overview

Mira is a cognitive-training game that combines *spatial memory* with *numerical reasoning*.
On each round, _n_ numbers appear at random positions on the screen.
You have a brief window to study them, then tap the first tile — at which point all remaining tiles go dark.
From then on, you must rely entirely on what you memorised to tap the rest in the correct order.

A wrong tap ends the round and resets your streak.

=== Why it is hard

* Numbers can be displayed in any base from 2 (binary) to 36 (hexatrigesimal), so the ordering is in that base's arithmetic, not necessarily decimal intuition.
* Complex, quaternion, and octonion tiles are ordered by *magnitude* (Euclidean norm), not by any individual coefficient — requiring you to mentally compute ‖z‖ before memorising positions.
* As the difficulty increases (auto-level mode), the tile count grows after every _k_ wins.

== Game Mechanics

=== The Memory Phase

[cols="1,3", options="header"]
|===
| Step | What Happens

| *Study*
| All _n_ tiles are visible with their numbers. Take as long as you like.

| *First Tap*
| Tap the tile with the smallest value (or smallest magnitude). All remaining unlabelled tiles go dark.

| *Recall*
| Tap the remaining tiles in ascending order from memory. The position of each tile is fixed; only the labels are hidden.

| *Win*
| All tiles tapped correctly → win dialog shows elapsed time, best time, and current streak.

| *Game Over*
| A wrong tile is tapped → streak resets to 0; a game-over dialog shows how far you got.
|===

=== Scoring

[cols="1,3", options="header"]
|===
| Metric | Description

| *Streak*
| Number of consecutive wins. Any wrong tap resets it to zero.

| *Time*
| Wall-clock time from first tap to last correct tap.

| *Best Time*
| Fastest win time for the current settings, persisted across sessions via `shared_preferences`.
|===

=== Auto-Level

When *Auto-level* is enabled, the game automatically increments the tile count (_n_) by 1 after every _k_ consecutive wins (configurable from 1 to 10).
The level resets when the tile count is changed manually.

== Number Types

Mira supports seven distinct number systems, selectable from Settings.

=== Sequential (`1 → n`)

Tiles are labelled 1, 2, 3, … _n_ in the chosen base.
Positions are shuffled; the ordering is trivially ascending.
Good for warming up or learning a new base.

=== Random Integer (`ℤ`)

Random integers drawn from the configured [min, max] range.
Displayed in the chosen base.
No two tiles share the same value.

=== Fraction (`p/q`)

Random reduced fractions with numerator in [min, max] and denominator in [2, denom_max].
GCD-reduced before display.
Ordering is by real value (_p/q_ as a float).

Example (base 10, denom_max = 6):

----
-3/4   1/2   5/3   -1/6   2
----

=== Real (`ℝ`)

Random floating-point values in [min, max], displayed to two decimal places.
Ordering is by real value.

=== Complex (`ℂ`)

Each tile displays a complex number _a + bi_.
Tiles are ordered by Euclidean magnitude ‖z‖ = √(a² + b²).

Coefficient values (a and b) are generated by one of four *coefficient types* (see <<coefficient-types>>).

Display format:

----
3+2i
-1+4i
----

=== Quaternion (`ℍ`)

Each tile displays a quaternion _a + bi + cj + dk_ across two lines.
Tiles are ordered by magnitude ‖q‖ = √(a² + b² + c² + d²).

Display format:

----
3+2i
+1j-4k
----

=== Octonion (`𝕆`)

Each tile displays an octonion with eight components across two lines using Unicode subscript unit labels (e₁ … e₇).
Tiles are ordered by magnitude ‖o‖ = √(Σᵢ aᵢ²).

Display format:

----
1+2e₁-3e₂+1e₃
+2e₄-1e₅+3e₆+0e₇
----

== Number Bases

Any number base from *2* to *36* can be chosen.
Digits above 9 are represented by uppercase letters A–Z.

[cols="1,2,1,2", options="header"]
|===
| Base | Name | Base | Name

| 2   | Binary           | 12  | Duodecimal
| 3   | Ternary          | 16  | Hexadecimal
| 4   | Quaternary       | 20  | Vigesimal
| 5   | Quinary          | 32  | Duotrigesimal
| 6   | Senary           | 36  | Hexatrigesimal
| 7   | Septenary        |     |
| 8   | Octal            |     |
| 9   | Nonary           |     |
| 10  | Decimal          |     |
| 11  | Undecimal        |     |
|===

NOTE: For hypercomplex types (complex, quaternion, octonion), coefficients are converted to the chosen base.
The real coefficients mode displays as decimal regardless of base since arbitrary real numbers do not have a clean base representation.

[[coefficient-types]]
== Coefficient Types

For complex, quaternion, and octonion numbers, each coefficient is generated independently using one of four modes:

[cols="1,3", options="header"]
|===
| Type | Description

| *Sequential* (`1..n`)
| Coefficients drawn from {1, 2, … _n_}, where _n_ is the tile count. Always positive.

| *Random Integer* (`ℤ`)
| Random integers in [coeffMin, coeffMax]. Zero is avoided (up to 50 retries).

| *Random Fraction* (`p/q`)
| Random reduced fractions with numerator in [coeffMin, coeffMax] and denominator in [2, coeffDenomMax].

| *Random Real* (`ℝ`)
| Random floats in [coeffMin, coeffMax], displayed to two decimal places.
|===

== Settings Reference

All settings are persisted between sessions.

[cols="2,1,3", options="header"]
|===
| Setting | Default | Description

| *Number Type*        | Sequential     | One of the seven number systems above.
| *Tile Count*         | 9              | Number of tiles per round (1–50).
| *Base*               | 10 (Decimal)   | Display base for integers and coefficients (2–36).
| *Min / Max*          | 1 / 50         | Range for random integer, fraction, and real types.
| *Denominator Max*    | 9              | Maximum denominator for fraction type.
| *Coefficient Type*   | Sequential     | Coefficient generation mode for hypercomplex types.
| *Coeff Min / Max*    | -5 / 5         | Range for random-integer and random-fraction coefficients.
| *Coeff Denom Max*    | 9              | Maximum denominator for fraction coefficients.
| *Auto-level*         | Off            | Automatically increase tile count after _k_ wins.
| *Wins per Level*     | 3              | How many consecutive wins before the tile count increments.
|===

== Implementation Notes

=== Tile Placement

Tiles are placed using a retry-based random algorithm (up to 400 attempts) that checks for overlap with already-placed tiles.
If the retry budget is exhausted, a uniform grid fallback guarantees all _n_ tiles always appear.

=== Tile Sizing

Tile dimensions are computed from the available screen area:

----
side = √(screenArea × density / n)
----

Where `density` and the maximum size cap vary by number type (octonions are largest; sequential integers are smallest).
The formula keeps the total tile area proportional to screen area regardless of tile count.

=== Font Sizing

The label font size is chosen heuristically from the line count and maximum line length of the label string, with a `FittedBox` as a safety net.
Multi-line labels (quaternion, octonion) use a smaller line height (`height: 1.35`).

=== Magnitude Uniqueness

For hypercomplex types, uniqueness is enforced by rounding each tile's magnitude to four decimal places and rejecting duplicates.
Up to 2000 attempts are made before falling back to whatever was successfully generated.

=== Negative Hexadecimal Ordering

Negative values in non-decimal bases are valid and correctly ordered.
For example, in base 16: −10₁₆ = −16₁₀ < −F₁₆ = −15₁₀, so the tile labelled `−10` sorts _below_ `−F`.
This is intentional — it is part of the challenge.

== Roadmap

=== Dual N-Back Integration

A planned extension embeds Mira's number-ordering sub-task inside each trial of a Dual N-Back working-memory paradigm:

* Each N-Back trial presents a grid cell position *and* a Mira tile (a number in a chosen system).
* The user must simultaneously track the N-back-ago position/number *and* the global ascending order of all numbers seen so far.
* Difficulty knobs: N value, number type, base, tile count per trial, inter-trial interval.

This combines visuospatial working memory (N-back position), auditory/verbal working memory (N-back number), and long-term serial-order recall (Mira ordering) into a single adaptive training session.
